# 博客写作规范

> 适用于 `docs/博客/` 下的所有博客。博客会公开发布，按本规范执行。
> 本项目（如意金股 / RuyiDailyStockAnalysis）是开源工具 daily_stock_analysis（AI 股票智能分析系统）的二开 fork。博客主要讲：这个工具是什么、怎么用、我们为什么二开、二开过程的开发记录与心得。整体就像**张大鹏在分享自己开发和维护这个项目的日常技术博客**。

## 位置与命名
- **博客成品统一放 `D:\notes\RuyiWriter\作品\博客\<年>年\<月>月\` 下**：一篇一个 `.md` 文件（扁平，**不要一篇一个子目录**）；图放该月共享的 `images\`，文件名加博客序号前缀防多篇撞名（如 `001_home.light.png`），正文用 `images/001_xxx.png` 引用。在 RuyiWriter 按年月集中管理；**本仓库不存放博客成品**，只保留本写作规范（`.claude/rules/blog.md`）。
- 文件名：`序号_标题.md`，三位序号（如 `001_如意金股：AI 自动出每日金股决策报告实录.md`）。
- 正文 H1 必须与文件名标题一致，不使用副标题。

## 标题规范（重点）
标题直接决定搜索与点击，逐条执行：

1. **每篇标题必须包含项目名「如意金股」**（铁令，不可省略）。
2. **长度约 20 字**（中文，含项目名后可略放宽），别太长，过长会被搜索引擎截断，也显啰嗦。
3. **搜索关键词前置**：高搜索量的词靠前。本项目强相关词：`AI 选股`、`A股`、`大模型`、`量化分析`、`自动盯盘`、`每日复盘`、`开源`、`二开`。一个标题最好能组合出多个长尾词。
4. **结尾带钩子**，三选一：疑问句引好奇 / 反差痛点前置 / 故事感。
5. **可适度借力背书/触发词**：GitHub star 数、榜单、真实推送效果等，但**适度，严禁关键词堆砌**，可读性第一。
6. **不做标题党**：标题承诺的，正文必须兑现。**严禁任何投资收益承诺或荐股暗示**，这是金融内容的红线。

> 定标题前的标准动作：先参考同领域爆款标题规律，拟 3 到 4 个候选让用户挑，不直接拍板。

## 开场白（每篇必须）
每篇博客正文的**第一句**（标题之后、正文开头），固定用口头禅开场：

> OK，OK，大家好，欢迎大家来到大鹏 AI 教育，我是张大鹏。

紧接着再进入正文。语气自然口语，像开直播 / 开课时跟观众打招呼那样。

## 内容与口吻
- **第一人称、第一视角**，全程以**我**（张大鹏）的身份分享**所见所闻**，像专家一样讲，有观点、有判断，不堆砌套话。
- **贴近真实场景举例**：多用直播间、课堂、给学员讲解、看回放复盘这类我真实会经历的场景来打比方，让抽象的东西落地、有画面感。
- **简体中文**。
- **所有技术论断必须对照真实代码核实**，不臆测、不照搬 README 概括；引用文件路径要准，优先用本仓库 `src/`、`data_provider/`、`strategies/` 下的真实实现佐证。
- **金融合规红线**：只讲技术与工具用法，**不预测涨跌、不荐股、不承诺收益**；涉及分析结果一律说明仅供学习参考、不构成投资建议。
- **发表级质量**：避免 AI 味（空泛排比、套话、emoji 堆砌）。
- 面向读者的正文**禁破折号 `——`、禁直角引号**，需要强调一律用加粗。

## 配图
- **每篇最多 3 张插图**（宁缺毋滥，图是为讲清内容服务的）；封面不单独做，从这 3 张插图里自选。
- 图放 `docs/博客/images/`，正文用相对路径 `images/xxx.png` 引用。
- 优先用项目真实截图（前端页面、决策报告、运行日志），别用无关配图。

## 写作方法论（复用 RuyiWriter）
具体写作手法不在这里重复造轮子，复用自媒体写作工作台 RuyiWriter 的方法论：

- 账号定位与口吻：`D:\notes\RuyiWriter\docs\账号定位.md`
- 提示词框架、起标题、套路化、模块化长文、去 AI 腔、多平台改写：`D:\notes\RuyiWriter\docs\知识库\`

**分层原则**：**通用「怎么写」** 看 RuyiWriter；**本项目专属**（放哪、命名、开场白、标题含项目名、金融合规、二开定位）**以本文件为准**，冲突时本文件优先。长文先出大纲、拟 3-4 个候选标题给用户挑，再动笔。
