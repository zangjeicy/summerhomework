# 课程版本基线方案

## 最终版本模型

RuyiDailyStockAnalysis 课程采用累积式演进，学生按版本顺序学习：

```text
v0.1.0  基础课程版本，所有学生从这里开始
   ↓ 补充、修复与小幅升级
v0.1.1  增量课程版本
   ↓ 新功能或较大架构演进
v0.2.0  下一阶段课程版本
```

后续版本只补充和升级，不覆盖、废弃或替代学生的起始版本。`v0.1.0` 已经是当前课程基线，不需要重新选择基线，也不得用新的年月 Course Edition 取代它。

## v0.1.0 的课程范围

以下三份主课件共同组成 `v0.1.0` 主课程体系：

1. `01 RuyiDailyStockAnalysis 零基础 AI 量化入门`
2. `02 RuyiDailyStockAnalysis 核心概念与项目原理`
3. `03 RuyiDailyStockAnalysis 功能与页面详解`

三份 PPT 可以逐步完善、验收和发布，不要求同时完成。现在补做 PPT 02、PPT 03，是补齐 `v0.1.0` 的完整课程体系，不代表它们属于 `v0.1.1`。

`v0.1.0` 还包括：

- 稳定教学代码包 `RuyiDailyStockAnalysis_v0.1.0.zip`
- 对应 docs、练习、截图、讲师备注、学生资料
- 问题记录、测试和验收证据
- 课程资产清单及必要的制作批次、PPT 修订号等元数据

## 三类版本不得混用

| 类型 | 示例 | 含义 |
| --- | --- | --- |
| 课程版本 | `v0.1.0`、`v0.1.1`、`v0.2.0` | 学生学习顺序和课程资产版本 |
| 教学代码包 | `RuyiDailyStockAnalysis_v0.1.0.zip` | 某课程版本对应的稳定、可复现代码真相源 |
| 开发仓库/产品版本 | `v3.24.1`、`3.25.0`、`dev@605fd015` | 上游产品发布和持续开发状态 |

三者可以通过 manifest 建立映射，但不能共用同一个版本含义。制作批次、构建编号或 PPT 修订号只能作为元数据，不能取代课程版本号。

## dev 与后续课程版本

`dev` 是后续课程版本的开发线，不是 `v0.1.0` 的替代品。

- `v0.1.0` PPT 必须按教学 ZIP 中的真实实现和限制讲解。
- 在备课、授课和学生使用中发现的问题，先记录为 `v0.1.0` 已知问题或待核验问题。
- 需要代码优化的事项进入 `dev`，完成修复、测试和验证后，成为 `v0.1.1` 或后续版本候选内容。
- `dev` 已有但 ZIP 不具备的功能，不得写成 `v0.1.0` 当前已有功能。
- `v0.1.0` 可以展示 `dev` 改进方向，但必须标记为“当前版本限制”“下一版本计划”或“下一版本预览”。

## 后续版本采用增量课程

`v0.1.1` 原则上不重做完整 PPT 01、02、03，可以新增：

- `v0.1.1 版本升级说明`
- `v0.1.1 已知问题修复`
- `v0.1.1 页面与功能变化`
- `v0.1.1 代码优化与架构演进`
- `从 v0.1.0 升级到 v0.1.1`
- 新功能专项 PPT

每个课程版本永久保留自己的 PPT、起始代码、最终代码、ZIP、docs、问题记录、练习、截图、测试和验收证据。学生先完成 `v0.1.0`，再学习后续增量版本。

## 每个课程版本建议记录和冻结的资产

manifest 用于描述和验证已经存在的课程版本资产，不用于重新发明版本体系。

| 资产 | 建议记录内容 |
| --- | --- |
| 课程身份 | 课程版本、制作批次、PPT 修订号、发布日期 |
| 教学代码 | ZIP 路径、大小、SHA-256、起始/最终代码说明 |
| Git 映射 | 已知仓库、branch/tag/commit；未知项明确写“暂未确认” |
| 依赖 | Python 版本、`requirements.txt`、Web/Desktop lockfile |
| 配置 | `.env.example`、课程最小配置、密钥安全说明 |
| 数据库 | schema/migration；需要时提供脱敏、可重建快照 |
| 演示数据 | 来源、日期、许可/脱敏、固定输入与预期输出 |
| 测试 | 后端、Web、关键页面/API、专项真实性核验结果 |
| 页面截图 | 路由、状态、数据来源、截图时间、对应教学 ZIP |
| docs | 架构、配置、API、验收、问题和升级说明 |
| PPT | 主课件文件名、哈希、页数、生成源、模板、字体和修订记录 |

缺少某个 manifest 字段时，应记录缺口并持续补证；不能因此否定既有课程版本。

## 主课程与授课副本

RuyiDailyStockAnalysis 主课程 PPT 是课程内容的唯一真相源。北京科技大学及其他学校、企业培训版本只能从主课程复制：

```text
RuyiDailyStockAnalysis 主课程 PPT
        ↓ 复制
北京科技大学等授课副本
        ↓
替换 Logo / 学校 / 讲师 / 日期 / 少量品牌元素
```

授课副本不得独立演进知识内容；课程内容修改必须先回到主课程版本。`09 零基础 AI 量化入门.pptx` 作为早期北科大授课副本和历史课程资产只读保留，不再承担未来主课程唯一真相源的角色。

## 归档与升级纪律

- `v0.1.0` 永久保留，不覆盖、不废弃、不重命名。
- 后续版本通过升级说明关联上一版本，清楚说明改了什么、为什么、如何修改、如何测试，以及页面和架构变化。
- tag、打包、提交和推送必须由独立任务和明确授权执行。
- 当前任务只修订版本规则，不创建任何新课程版本。
