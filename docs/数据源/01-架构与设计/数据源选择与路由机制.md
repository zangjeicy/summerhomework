# 数据源选择与路由机制

## 日线行情

日线入口是 `DataFetcherManager.get_daily_data`。候选数据源先按实例 `priority` 排序，再根据市场和运行可用性过滤。

当前市场支持表位于 `DataFetcherManager._DAILY_MARKET_FETCHER_SUPPORT`：

- A 股：EFinance、Tencent、AkShare、Tushare、TickFlow、Pytdx、Baostock、YFinance。
- 港股：AkShare、Tushare、YFinance、Longbridge。
- 美股：YFinance、Longbridge、Finnhub、Alpha Vantage。
- 日本、韩国、台湾：当前表中仅 YFinance。

美股日线存在显式顺序：

- 美股指数：YFinance 在前，随后 Finnhub。
- 启用 Longbridge 优先时：Longbridge、Finnhub、Alpha Vantage、YFinance。
- 默认美股个股：Finnhub、Alpha Vantage、YFinance、Longbridge。

其他市场主要依赖初始化后的 `priority` 顺序。

## 日线失败切换

每个候选源执行统一的 `get_daily_data`。成功时记录健康状态并返回；失败时记录诊断和健康失败，然后继续下一个候选源。所有候选源失败时才向上抛出数据获取错误。

这里的 fallback 是“换供应商”，不是同一供应商内部的 HTTP 重试。具体 Fetcher 还可能通过 `tenacity` 实现自己的重试，两者必须区分。

## 实时行情

实时入口是 `DataFetcherManager.get_realtime_quote`，默认配置位于 `Config.realtime_source_priority`，当前默认值为：

```text
tencent,akshare_sina,efinance,akshare_em
```

实际候选还会根据市场改变：

- 港股代码走港股专用转换与 AkShare 路径，避免扇出到 A 股数据源。
- 美股会加入 YFinance、Longbridge、Finnhub、Alpha Vantage 等可用源。
- TickFlow 配置后可以加入实时优先级。

实时源名称是配置别名，不总与 Fetcher 类名一一对应。例如 `akshare_sina` 和 `akshare_em` 都由 AkShare Fetcher 以不同 `source` 参数完成。

## 基本面

管理器初始化两个适配器：

- A 股使用 `AkshareFundamentalAdapter`。
- 港股和美股使用 `YfinanceFundamentalAdapter`。

基本面包含独立缓存和预算控制，不能按日线数据源优先级推断基本面来源。

## 新闻

新闻入口在 `SearchService.search_stock_news`。搜索供应商、查询变体、时间窗口、严格新鲜度和直接公司新闻优先等逻辑位于 `src/search_service.py`。新闻 fallback 与行情 Fetcher 管理器是两套独立机制。

## 审查要点

- 先确认能力类型，再讨论数据源优先级。
- 先确认市场识别结果，再判断候选源。
- 区分供应商内部重试、跨供应商 fallback 和健康熔断。
- 不要仅根据 `data_provider/__init__.py` 顶部说明判断最终顺序；以管理器当前代码和配置为准。
