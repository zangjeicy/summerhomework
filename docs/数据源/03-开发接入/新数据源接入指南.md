# 新数据源接入指南

## 接入前判断

先确认供应商实际支持的市场、实时行情、历史日线、复权、基本面、新闻和调用限制。不要因为 SDK 有某个方法就直接声明项目已支持该能力。

## 日线接入步骤

1. 在 `data_provider/` 新增 Fetcher，并继承 `BaseFetcher`。
2. 实现 `_fetch_raw_data(stock_code, start_date, end_date)`。
3. 实现 `_normalize_data(df, stock_code)`，输出标准日线列。
4. 明确供应商股票代码格式，复用现有市场识别和代码转换工具。
5. 设置类名 `name` 和 `priority`，但不要只依赖优先级表达市场能力。
6. 在 `DataFetcherManager._init_default_fetchers` 中按配置条件注册。
7. 在 `_DAILY_MARKET_FETCHER_SUPPORT` 中声明已验证市场。
8. 如需改变特定市场顺序，在 `DataFetcherManager.get_daily_data` 的显式路由中处理。
9. 为原始字段转换、空数据、异常和市场代码编写离线测试。
10. 将真实网络验证放入标记明确的 smoke test，不让默认测试依赖密钥。

## 实时行情接入

实时行情还需：

- 实现 `get_realtime_quote`。
- 使用或转换为 `UnifiedRealtimeQuote` 的公共字段。
- 在实时别名映射和配置优先级解析中注册。
- 说明行情时间、交易所时区和陈旧判断依据。
- 测试市场专用路由，避免港股或美股请求误入 A 股供应商。

## 配置要求

新增配置必须满足：

- 未配置时系统仍能运行。
- 密钥只从环境变量或现有配置服务读取。
- 同步更新 `.env.example`。
- 日志不得输出密钥。
- 初始化失败不能阻断其他可用数据源。

## 错误语义

- 网络错误、限流和供应商返回错误应保留可诊断摘要。
- 空结果只有在供应商明确允许空数据的场景才能当作成功。
- 不允许用宽泛的 `except` 直接返回空列表来掩盖契约错误。
- 如果发生 fallback，诊断应能显示来源和目标。

## 完成定义

新数据源只有同时具备实现、配置示例、离线测试、能力矩阵更新、接入文档更新和必要的网络 smoke test，才算完成接入。
