# 通用数据契约

## 日线标准列

`BaseFetcher` 要求具体数据源把原始响应转换为：

- `date`：可转换为 `datetime` 的日期。
- `open`：开盘价。
- `high`：最高价。
- `low`：最低价。
- `close`：收盘价。
- `volume`：成交量，单位仍需以具体供应商与适配器说明为准。
- `amount`：成交额，供应商不提供时可能缺失。
- `pct_chg`：涨跌幅，具体数值是百分数还是比例必须由适配器保证一致。

清洗阶段会把数值列用 `pandas.to_numeric(errors="coerce")` 转换，并删除 `close` 或 `volume` 为空的行。

## 派生字段

统一日线入口继续计算：

- `ma5`、`ma10`、`ma20`。
- `volume_ratio`。

项目当前的 `volume_ratio` 是日线成交量相对于前期均量的倍数，不是交易软件常见的同一分时时刻量比。课程和博客不得混用两种口径。

## 实时行情

实时行情的公共类型位于 `data_provider/realtime_types.py`。审查时至少确认：

- 股票代码与名称。
- 当前价、昨收、涨跌额、涨跌幅。
- 成交量和成交额。
- 行情时间。
- 真实数据源。

不得用抓取时间替代行情时间，也不得因为返回字典非空就认定数据仍然新鲜。

## 基本面状态包

基本面适配器返回带 `status` 的数据包。现有测试覆盖 `partial` 和 `not_supported` 等状态。调用方必须先读状态，再使用字段；不能把缺失字段自动解释为零。

## 缺失值原则

当前各链路尚未完全统一，新增适配器至少遵守：

- “供应商不支持”与“请求失败”不得混为一谈。
- “返回成功但没有记录”不得伪造成全零记录。
- 未知数值优先保留为空并附带状态，不自行估算。
- 估算字段必须显式标记，不能与供应商原始字段混合后失去来源。
