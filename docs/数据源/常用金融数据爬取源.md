# 常用金融数据爬取源

> 更新日期：2026-07-17  
> 适用范围：RuyiDailyStockAnalysis 的行情、基本面、宏观和资讯数据选型。本文所称“爬取源”同时包括官方 API、授权数据服务、开源聚合库和公开网页接口；它们在稳定性与使用授权上并不等价。

## 1. 先看结论

金融数据接入不应只追求“免费可调用”。生产流程更关心数据授权、字段口径、复权方式、交易日历、更新时间、限流和故障切换。

本项目建议采用分层组合：

1. A 股研究主源优先使用有 Token、字段定义和配额规则的 Tushare；AKShare、EFinance、腾讯、Pytdx、Baostock用于补充或降级。
2. 港美股主源优先考虑 Longbridge 等有明确行情权限的服务；YFinance适合个人研究和故障兜底。
3. 美股基本面原始事实可补充 SEC EDGAR，宏观数据可补充 FRED。
4. 不把单个网页接口当成唯一生产数据源；所有来源都应设置超时、重试上限、缓存、限速和来源诊断。
5. “接口能返回数据”不代表数据可用于商业分发。上线前必须复核供应商条款和交易所行情授权。

## 3. A 股与中国市场常用来源

### 3.1 Tushare Pro

- 官方文档：[Tushare 数据接口](https://tushare.pro/document/2)
- 访问方式：Python SDK或 `https://api.tushare.pro` HTTP API，需要 Token。
- 典型数据：证券列表、交易日历、日线/分钟/Tick、复权因子、每日指标、三大财务报表、业绩、分红、资金流和部分新闻。
- 代码格式：以 `ts_code` 表示交易所，例如 `600519.SH`、`000001.SZ`、`00001.HK`。
- 优点：字段定义相对清晰，适合批量拉取和本地增量库；财务与行情可用统一代码关联。
- 注意：接口权限由积分或单独授权决定，单次返回行数和调用频率因接口而异，不能只用一次小查询判断所有接口都可用。

在本项目中，配置 `TUSHARE_TOKEN` 后会初始化 `TushareFetcher`，并使用项目内部限速器。建议把交易日历、日线、复权因子和基础资料分别做缓存，不要在每次单股分析中重复全量拉取。

### 3.2 AKShare

- 官方文档：[AKShare 数据字典](https://akshare.akfamily.xyz/data/index.html)
- 访问方式：Python 包，多数接口无需 Key。
- 典型数据：A/H/美股行情，基金、期货、期权、债券、外汇、宏观、行业板块、公司信息和新闻等。
- 优点：覆盖范围很广，接口名称直接对应业务数据集，适合研究原型和补齐长尾字段。
- 注意：AKShare是聚合访问层，具体接口可能来自东财、新浪、腾讯、雪球或交易所网页。上游页面、参数、反爬策略改变时，接口会暂时失效或字段变化。

建议固定依赖版本，对关键接口做字段契约测试；日志中同时记录“AKShare接口名”和“真实上游来源”，避免把聚合库名称误当成原始数据出处。

### 3.3 EFinance（东方财富聚合）

- 项目主页：[Micro-sheep/efinance](https://github.com/Micro-sheep/efinance)
- 访问方式：Python 包，无 Key。
- 典型数据：股票历史行情、实时行情，以及基金、债券和期货数据。
- 优点：中文字段直观，上手快，适合个人研究和轻量任务。
- 注意：项目文档明确仅供学习交流，并提示商业用途限制；其主要行情路径依赖东方财富。维护者曾说明东方财富自2025年起加强 IP 访问频率限制，批量请求容易出现断连或封禁。

应优先使用批量接口和本地增量缓存，禁止高并发逐股票刷新。出现 `RemoteDisconnected`、HTTP 频控或空结果时，应立即切换来源，不要无限重试。

### 3.4 腾讯、新浪、东方财富网页接口

这类来源常被 AKShare 或项目适配器封装，可提供实时报价、K 线、资金流或板块数据。它们通常不是面向第三方长期承诺的正式开放 API。

- 优点：无需 Key、延迟低、适合作为实时行情降级路径。
- 风险：参数和返回格式可能无通知变化；Cookie、Referer、签名、限流和地域策略会改变；网页可访问不等于允许批量采集或再分发。
- 建议：仅放入带熔断的 fallback 链；保留原始响应样本和字段映射测试；避免绕过验证码、登录、付费墙或技术访问控制。

### 3.5 Pytdx

- 文档：[Pytdx 标准行情 API](https://pytdx-docs.readthedocs.io/zh-cn/latest/pytdx_hq/)
- 访问方式：连接通达信行情节点，Python 客户端请求行情、K线和部分公司资料。
- 优点：无需网页解析，适合A股低成本行情补充。
- 注意：需要维护可用节点列表、连接超时和断线重连；不同节点的速度和稳定性不同。使用前还应核实协议及行情数据的授权要求。

### 3.6 Baostock

- 官方站点：[Baostock](https://baostock.com/)
- 访问方式：Python 包登录会话，无需注册 Token。
- 典型用途：A股历史K线、复权数据、证券基础信息、财务指标和部分宏观数据。
- 优点：适合教学、离线研究和历史数据兜底。
- 注意：不应作为盘中实时行情主源；需要检查交易日、复权参数、停牌数据和最新数据日期。

## 4. 港股、美股与全球市场来源

### 4.1 YFinance

- 文档：[yfinance documentation](https://ranaroussi.github.io/yfinance/)
- 访问方式：Python 包，无 Key，读取 Yahoo Finance 公开接口。
- 典型数据：历史行情、公司信息、财务报表、公司行动、期权链、ETF持仓和市场摘要。
- 优点：全球代码覆盖广，研究代码简单，适合港美股和跨市场兜底。
- 关键限制：yfinance明确声明它不是 Yahoo 官方产品，Yahoo Finance 数据主要面向个人使用；分钟数据历史窗口有限，例如文档说明盘中数据通常不能超过最近60天。

生产使用时要设置 timeout、时区规范、复权口径和缓存，并对 `info` 等可能频繁变化的字段做缺失容错。

### 4.2 Longbridge OpenAPI

- 官方文档：[Longbridge Developers](https://open.longbridge.com/docs)
- 访问方式：开户并申请 OpenAPI 权限，SDK支持 HTTP 与 WebSocket。
- 覆盖：A/H/美股行情，K线、逐笔、深度、期权/窝轮，以及公司资料、财务、估值和新闻；交易能力应与只读行情能力隔离。
- 频率：官方概览说明行情接口不超过每秒10次且并发不超过5，SDK会对行情请求主动限速；实际权限和配额以账号页面为准。
- 行情权限：OpenAPI行情权限与 App/Web 权限分开，港股基础行情可能为延迟行情，实时与深度行情需相应权限。

这是本项目港美股高质量实时行情的优选来源之一。配置时只授予行情所需权限，分析服务不应持有交易权限或下单凭据。

### 4.3 Finnhub

- 官方站点：[Finnhub Stock API](https://www.finnhub.io/)
- 访问方式：API Key，REST与WebSocket。
- 典型数据：美股实时行情、公司资料、新闻、财务、分析师预期和另类数据。
- 优点：接口结构化，适合美股实时行情和基本面补充。
- 注意：免费层与商业层在市场范围、历史深度、调用频率和授权上差异明显；接入前应按当前价格页核对配额和使用许可。

### 4.4 Alpha Vantage

- 官方文档：[Alpha Vantage API Documentation](https://www.alphavantage.co/documentation/)
- 访问方式：API Key。
- 典型数据：全球股票时序、外汇、数字货币、商品、经济指标、技术指标、公司基本面和新闻情绪。
- 优点：REST/JSON/CSV易于接入，适合轻量全球市场补充。
- 注意：实时、大批量、完整历史和部分高级数据属于付费能力；免费层应严格缓存并控制调用频率。不要把技术指标 API 返回值与本项目本地指标计算混用而不记录口径。

## 5. 官方基本面与宏观补充源

### 5.1 SEC EDGAR

- 官方说明：[EDGAR API Overview](https://www.sec.gov/file/api-overview)
- 典型数据：公司提交历史、10-K/10-Q/8-K等申报文件、XBRL Company Facts和批量归档。
- 优点：美国上市公司监管申报的权威原始来源，适合核验财务事实和重大事项。
- 注意：它不是实时行情接口。客户端应携带能识别组织和联系方式的 `User-Agent`，遵守SEC公平访问策略，控制请求频率并优先使用批量文件和本地缓存。

### 5.2 FRED 与 ALFRED

- 官方文档：[FRED API](https://fred.stlouisfed.org/docs/api/fred/)
- 典型数据：利率、通胀、就业、货币、信用、产出等宏观时间序列。
- FRED适合读取当前发布历史；ALFRED可用于研究数据在不同历史时点的修订版本，避免回测中的前视偏差。
- 注意：宏观序列频率、单位、季调方式和发布日期必须随数据一起保存，不能只保存数值。

## 6. 新闻与公告抓取

新闻数据应按可信度分层：

1. 交易所、监管机构和上市公司公告属于一级事实源。
2. 授权新闻 API 或供应商结构化新闻属于二级聚合源。
3. 搜索引擎、RSS和财经网页只用于发现线索，必须保留原链接、发布时间、抓取时间和来源名称。

不要把搜索摘要直接当作公告正文，也不要绕过登录、订阅、验证码、robots或其他访问限制。新闻接口返回零条时应标记 `degraded`，不能解释为“确认没有利空”。

## 7. 生产级抓取规范

### 7.1 每条记录至少保存的元数据

- `provider`：适配器或供应商，例如 `tushare`。
- `upstream_source`：真实原始来源，例如交易所、东财或 Yahoo。
- `symbol` 与 `market`：标准化后的代码及市场。
- `data_time`：数据代表的交易时间。
- `fetched_at`：实际抓取时间，必须包含时区。
- `adjustment`：不复权、前复权或后复权。
- `currency`、`unit`、`timezone`。
- `quality_status`：`ok`、`partial`、`stale`、`degraded`或`failed`。

### 7.2 调用策略

- 使用连接池、明确的连接/读取超时和有限次指数退避。
- 对429、403、验证码、HTML替代JSON、字段缺失和空结果分别分类，不能统一吞成空DataFrame。
- 对交易日历、证券列表、财务报告做长缓存；对日线做按交易日增量更新；实时行情使用短缓存或推送。
- 限速应按域名、账号和接口分别控制。多个适配器共享同一上游域名时，必须共享限速预算。
- 主源失败后执行有序降级，并在最终报告中披露实际使用来源和缺失字段。
- 保存原始响应哈希或脱敏样本，便于字段漂移审计；严禁在日志、报告或缓存键中泄露 Token。

### 7.3 数据校验

- OHLC关系：`low <= open/close <= high`。
- 成交量、成交额和换手率的单位与缩放。
- 交易日、停牌、除权除息和部分交易日K线。
- 实时价与最新日线的时间边界，盘中不能把未收盘K线冒充完整日线。
- 多源交叉检查时先统一复权、币种、时区和交易时段，再比较价格差异。

## 8. 在本项目中的推荐路由

| 场景 | 推荐主源 | 推荐降级链 | 说明 |
| --- | --- | --- | --- |
| A股日线 | Tushare（已配置且有权限） | EFinance → AKShare → Pytdx → Baostock → YFinance | 实际顺序仍受项目配置和健康熔断影响 |
| A股实时行情 | EFinance/腾讯/AkShare | Tushare或其他可用源 | 网页上游容易限流，应共享缓存 |
| 港股日线/实时 | Longbridge | AKShare → YFinance | 区分延迟与实时行情权限 |
| 美股日线/实时 | Longbridge/Finnhub | Alpha Vantage → YFinance | 免费层通常不等于完整实时授权 |
| A股基本面 | Tushare或AKShare适配器 | 缺失时显式降级 | 不应由行情字段推断财务数据 |
| 港美股基本面 | Longbridge/Finnhub | YFinance；美股原始事实用EDGAR核验 | 财报发布日期和币种必须保留 |
| 宏观上下文 | 官方统计源/FRED | AKShare宏观聚合 | 回测优先使用带历史修订的数据 |
| 新闻公告 | 交易所/监管/公司公告 | 授权新闻API → RSS/搜索发现 | 搜索无结果不等于无事件 |

新增数据源时请继续阅读[新数据源接入指南](03-开发接入/新数据源接入指南.md)、[通用数据契约](02-数据契约/通用数据契约.md)和[数据源代码审查清单](05-测试与审查/数据源代码审查清单.md)。

## 9. 维护说明

- 第三方接口、免费配额、行情权限和使用条款会变化；每次升级依赖或上线商业功能前重新核对官方文档。
- 文档中的“推荐”是工程选型建议，不代表投资建议，也不替代数据供应商授权。
- 当文档与代码行为不一致时，以可执行代码和真实运行诊断为准，并同步修正文档。
