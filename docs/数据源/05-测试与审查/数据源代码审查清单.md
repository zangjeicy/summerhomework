# 数据源代码审查清单

## 必要性与范围

- [ ] 新供应商解决了明确的市场、稳定性或字段缺口。
- [ ] 改动没有夹带无关路由重构。
- [ ] PR 描述区分供应商能力与项目实际接入能力。

## 市场与代码

- [ ] 复用了现有市场识别和股票代码转换逻辑。
- [ ] A股、港股、美股及指数代码有代表性测试。
- [ ] 不支持的市场会在调用前跳过，而不是请求后猜测。

## 数据契约

- [ ] 日线输出符合标准列。
- [ ] 日期、数值、成交量和成交额单位有明确说明。
- [ ] 复权口径明确，未默认为供应商的未知行为。
- [ ] 缺失值没有被静默改成零。
- [ ] 估算字段带有显式标记。

## 稳定性

- [ ] 设置了合理超时。
- [ ] 重试只覆盖适合重试的异常。
- [ ] 限流不会形成无限重试。
- [ ] 单一供应商失败不会阻断其他候选源。
- [ ] 异常没有被宽泛捕获并吞成空结果。
- [ ] fallback 后能够识别最终来源。

## 配置与安全

- [ ] 密钥不写死。
- [ ] `.env.example` 已同步。
- [ ] 未配置密钥时仍能启动。
- [ ] 日志和测试夹具不包含真实密钥。

## 测试与文档

- [ ] 原始字段转换具有离线单元测试。
- [ ] 空数据、超时、异常和 fallback 有回归测试。
- [ ] 网络 smoke test 有明确标记和超时。
- [ ] 能力矩阵和接入指南已更新。
- [ ] `docs/CHANGELOG.md` 的 `[Unreleased]` 使用扁平条目。
