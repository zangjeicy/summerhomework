# 北科大 2026 · 上市公司数据示例

本目录以 28 列的完整上市公司数据为唯一教学主数据。原先的 2 列精简数据及旧脚本保存在 `legacy/`，仅用于历史追溯，不再作为运行入口。

## 目录结构

- `01_获取上市公司完整信息.py`：联网重新生成完整主数据。
- `02_补充上市公司缺失数据.py`：只重查失败记录，覆盖主数据前自动备份。
- `03_根据股票代码查询公司信息.py`：按股票代码查询全部字段。
- `04_根据关键字查询公司信息.py`：按名称关键字查询并导出全部字段。
- `05_验证tushare是否可用.py`：读取项目根目录 `key.txt`，真实调用交易日历接口验证 Tushare Token。
- `06_tushare的综合应用.py`：组合基础信息、交易日历、日线、复权因子、每日指标和资金流向，生成综合数据 CSV。
- `data/上市公司.csv`：当前完整主数据，5529 行、28 列。
- `data/backup/`：完整数据快照及后续更新前备份。
- `output/`：关键字查询等派生结果，可随时由主数据重新生成。
- `legacy/`：旧精简版和旧完整版脚本，不参与标准流程。

## 常用命令

```powershell
uv run python examples/北科大2026/03_根据股票代码查询公司信息.py 600519
uv run python examples/北科大2026/04_根据关键字查询公司信息.py 银行
uv run python examples/北科大2026/02_补充上市公司缺失数据.py
uv run python examples/北科大2026/05_验证tushare是否可用.py
uv run python examples/北科大2026/06_tushare的综合应用.py
```

重新抓取全量数据耗时较长，日常查询直接复用 `data/上市公司.csv`。需要恢复基线时，将 `data/backup/上市公司_完整备份_20260716.csv` 复制为 `data/上市公司.csv`。
