# 自有 Release 发布通道

> 目标：读者从**本仓库**（DaPengRuYi/RuyiDailyStockAnalysis）的 Releases 下载含如意二开功能的客户端，而不是上游的。
> 状态（2026-07-04）：通道机制已验证可触发，**当前阻塞在 GitHub Actions 计费**（见「已知阻塞」）。

## 通道机制（上游已给全，无需改造）

推送一个 `v*.*.*` 格式的 tag 到本仓库，会自动触发两条 workflow：

| workflow | 触发 | 产物 |
|---|---|---|
| `create-release.yml` | tag push | 创建 GitHub Release + 自动生成 Release Notes |
| `desktop-release.yml` | tag push 或 `workflow_dispatch`（指定已有 tag） | Windows NSIS 安装包 + 免安装 zip + macOS x64/arm64 dmg，上传到该 tag 的 Release |

注意事项：

- tag 必须严格匹配 `^v\d+\.\d+\.\d+$`（`desktop-release.yml` 有正则校验，**不支持** `-ruyi.1` 之类后缀）。
- `auto-tag.yml` 是 opt-in：push 到 main 且 commit title 含 `#patch` / `#minor` / `#major` 才会自动打下一个版本 tag。
- 手动打 tag 必须用 annotated tag（AGENTS.md 约定）；tag push 属对外发布动作，**需大鹏确认后执行**。

## 版本号策略（自有 tag 与上游共存）

- 本地与 origin 的 tag 延续上游版本线（当前最新 `v3.24.1`，共 166 个 tag 已在 origin）。
- `upstream` remote 已配置 `tagOpt = --no-tags`，**永不拉取上游 tag**——此后上游即使发布同名版本（如它自己的 v3.25.0）也不会冲进本地，我们的版本线以 origin 为唯一真源。
- 自有发版从 `v3.25.0` 起（用 `#minor` 或手动 annotated tag），与上游已同步进来的 `v3.24.x` 自然衔接。

## 手动触发验证构建（不打新 tag）

```bash
gh workflow run desktop-release.yml --repo DaPengRuYi/RuyiDailyStockAnalysis -f release_tag=v3.24.1
gh run list --repo DaPengRuYi/RuyiDailyStockAnalysis --workflow=desktop-release.yml --limit 3
```

## 已知阻塞（2026-07-04 验证记录）

用 `workflow_dispatch` + `release_tag=v3.24.1` 实测触发成功（run 28693167468），三个构建 job（windows / macos-x64 / macos-arm64）全部在启动前失败，失败注解：

> The job was not started because recent account payments have failed or your spending limit needs to be increased.

原因：本仓库当前为**私有**，Actions 消耗付费分钟数（Windows 2 倍、macOS 10 倍计费），账户当前支付状态/spending limit 不允许。两条解法（待大鹏拍板）：

1. **仓库转公开**（推荐，符合生态规划的开源引流）：公开仓库标准 runner 分钟数免费；转公开前按生态规划完成检查——密钥清查（运行日志 `ea.txt`/`err.txt` 等已确认 gitignore 未入库）、README 二开声明、免责声明。
2. **保持私有 + 修复计费**：GitHub Settings → Billing 处理支付方式或上调 spending limit；注意每次全平台构建约消耗数百等效分钟。
